# HonePage – concept

## 	1.	Hero-блок
	•	CTA лучше один: например, «Войти», а под ним мелкой ссылкой «У меня нет доступа» → форма заявки/контакты (потом доделаем).
	•	Под слоганом можно добавить одну строку уточнения:
«Помогает флот-менеджерам находить оптимальную цену на каждую неделю сезона».

##	2.	“Как это работает”
	•	Шаги:
	1.	Выберите неделю и яхту
	2.	Запустите сканирование конкурентов
	3.	Сравните цены и скидки
	4.	Утвердите базовую цену на неделю
	3.	“Ключевые преимущества”
Формулировки ок, можно собрать в короткие bullet’ы с жирным заголовком:
	•	Интеграция с NauSYS и внутренней БД
	•	UI под задачи флот-менеджера, а не маркетинга
	•	История решений и изменение цен по неделям
	•	База для аналитики по сезону (пока без обещаний “дашбордов”, если их нет)

##	4.	“Для кого”
три целевые роли:
	•	Флот-менеджеры
	•	Владельцы яхт
	•	Чартерные компании

##	5.	CTA-блок внизу
Повторить главный месседж, например:
«Хотите управлять ценами системно, а не в Excel?»
Кнопка: «Запросить доступ» / «Связаться с нами» — если продукт пока не self-service.

##	6.	Footer
Всё ОК, можно добавить маленькую строчку:
«Pilot version · Internal use only» (если это пока пилот).
